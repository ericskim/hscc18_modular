Detailed Description

The transition function of the abstraction.

The transition function can either be read from file or constructed with the help of AbstractionGB.

The set of states is given where N is stored in m_no_states The set of inputs is given where M is stored in m_no_inputs

A transition is a triple where and

  • the element is referred to as pre
  • the element is referred to as post
  • the element is referred to as input

The data structure is particularly tailored to be used in the minimal and maximal fixed point computations, in which the list of pres i associated with a post k and input j need to be accessed fast:
the pres itself are stored in the array m_pre
the position of the pres in m_pre is stored in the array m_pre_ptr
the number of pres is stored in m_no_pre

To copy the pres of the state-input pair (k,j) to a std::vector<abs_type> pre the following code is implemented in get_pre(k,j) 

for(abs_type i=0; i<m_no_pre[k*M+j]; i++) {
  pre.push_back(m_pre[m_pre_ptr[k*M+j]]+i);
}

A transition function can only be moved and not copied.
